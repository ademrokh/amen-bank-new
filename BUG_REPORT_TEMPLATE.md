## 📋 Bug Report Severity Guide

### 🔴 CRITICAL
- System/site completely broken
- Cannot complete essential task
- Data loss or corruption
- Security vulnerability
- No possible workaround
- Affects all users
- **Example**: Contact form doesn't submit, chatbot completely unavailable

### 🟠 HIGH
- Core feature significantly impaired
- Major usability issue
- Performance severely degraded
- Accessibility barrier
- Workaround exists but difficult
- Affects significant user base
- **Example**: Mobile form buttons too small to tap, chatbot responses are incorrect

### 🟡 MEDIUM
- Feature works but degraded
- Inconsistent behavior
- Minor usability issue
- Performance slightly slower
- Affects subset of users
- Easy workaround available
- **Example**: Language switch is slow, button text slightly cut off on some devices

### 🟢 LOW
- Cosmetic issue only
- Inconsistent styling
- Minor grammatical error
- Affects very few users
- No functional impact
- No workaround needed
- **Example**: Button color doesn't match design mockup, typo in FAQ answer

## 🎯 What Makes a Good Bug Report?

✅ **GOOD**:
- Clear title: "Contact form email field rejects valid emails"
- Specific steps: 1. Go to contact page 2. Enter john@example.com 3. See error
- Reproducible: "Happens every time"
- Has screenshot showing the problem
- Includes device and browser info

❌ **POOR**:
- Vague title: "Something broken"
- Unclear steps: "Test the form"
- Can't reproduce: "It happened once"
- No screenshots
- Missing device info
